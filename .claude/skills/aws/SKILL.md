## Purpose
This skill provides a structured approach to debugging code, identifying issues, and resolving them efficiently. It is designed for developers working on complex projects.

## Workflow
1. **Understand the Problem**:
   - Reproduce the issue.
   - Gather error messages, logs, or unexpected behavior details.

2. **Isolate the Cause**:
   - Use debugging tools (e.g., breakpoints, logs).
   - Narrow down the code section causing the issue.

3. **Analyze the Code**:
   - Review the logic and dependencies.
   - Check for common issues (e.g., null references, incorrect conditions).

4. **Test Fixes**:
   - Apply potential solutions.
   - Test changes in a controlled environment.

5. **Verify and Document**:
   - Ensure the issue is resolved.
   - Document the root cause and solution.

## Examples
- "Debug a null pointer exception in Java."
- "Identify why a Python script is running slowly."

## Decision Points
- When to escalate the issue.
- Whether to refactor code for long-term fixes.

## Quality Checks
- Ensure the fix does not introduce new issues.
- Verify the solution aligns with best practices.
